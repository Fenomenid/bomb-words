# Слова-мины

Онлайн-игра для компании: сайт ведет комнаты, роли, таймеры, мины и очки.

## Как играть

1. Хост создает комнату, выбирает настройки и отправляет ссылку игрокам.
2. В раунде один игрок объясняет слово, следующий по кругу угадывает, остальные становятся минерами.
3. Минеры заранее добавляют и редактируют слова-мины. Объясняющий и отгадывающий список мин не видят.
4. Когда все мины готовы или таймер на мины вышел, начинается объяснение.
5. Минеры во время объяснения отмечают сработавшие мины. Отметку можно снять, если нажали случайно.
6. Если слово угадали, объясняющий подтверждает это кнопкой `Угадали`.
7. Отгадывающий может сдаться, а объясняющий может завершить раунд как неугаданный.
8. За угаданное слово объясняющий и отгадывающий получают по 1 очку.
9. За провал или таймер объясняющий и отгадывающий теряют по 1 очку.
10. Каждая сработавшая мина дополнительно снимает по 1 очку с объясняющего и отгадывающего, а автору мины дает +1.
11. После результата следующий раунд запускается автоматически, пока не выполнено условие финала.

## Возможности

- Комнаты по приглашению.
- Realtime-синхронизация через Socket.IO.
- Разное состояние для ролей: секретные мины не отправляются объясняющему и отгадывающему.
- Настройки комнаты: сложность, количество мин, таймеры, финал по очкам или раундам.
- Уровни сложности: легкий, средний, нереальный.
- Свой словарь для хоста: одно слово на строку, хранится только в памяти комнаты.
- Темная и светлая тема.
- Звук последних секунд таймера с переключателем.
- Пауза таймера для хоста.
- Итоговый экран с топ-3.

## Стек

- Frontend: React, Vite, TypeScript.
- Backend: Node.js, Express, Socket.IO, TypeScript.
- Хранение: in-memory.

## Render

Проект рассчитан на деплой как один Node Web Service.

```text
Build Command: npm install && npm run build
Start Command: npm start
```

Environment:

```text
NODE_ENV=production
```

Render задает `PORT` автоматически. В production сервер отдает собранный `client/dist`, а клиент подключается к Socket.IO на том же домене.

## Ограничения

- Комнаты и очки хранятся в памяти процесса.
- После перезапуска сервера состояние сбрасывается.
- При отключении игрок удаляется из комнаты.
- Если отключился хост, хостом становится следующий игрок.
- Если отключился объясняющий во время раунда, раунд завершается как пропущенный.
