# Слова-мины

MVP онлайн-игры с комнатами и realtime-логикой. Голос и видео не встроены: игроки общаются во внешнем созвоне.

## Стек

- Frontend: React, Vite, TypeScript, Socket.IO Client
- Backend: Node.js, Express, Socket.IO, TypeScript
- Хранение: in-memory, без базы данных

## Запуск локально

Установить зависимости:

```bash
npm install
```

Запустить клиент и сервер:

```bash
npm run dev
```

Открыть клиент:

```text
http://localhost:5173
```

Сервер работает на:

```text
http://localhost:3001
```

## Как проверить MVP

1. Откройте `http://localhost:5173`.
2. Введите имя и создайте комнату.
3. Скопируйте ссылку приглашения и откройте ее во втором и третьем браузере или в инкогнито-окнах.
4. Войдите разными именами.
5. Хост нажимает "Начать игру".
6. В лобби хост может настроить количество мин и таймеры.
   Также хост выбирает финал: по целевым очкам или по количеству раундов.
7. Объясняющий видит слово, но не видит список мин.
8. Минеры добавляют мины и видят список. Когда таймер мин заканчивается, объяснение стартует автоматически.
9. Раунд можно завершить кнопками "Угадали", "Скип" или дождаться таймера. Минеры нажимают на сработавшие мины во время объяснения.
10. После результата следующий раунд стартует автоматически по таймеру, либо хост запускает его вручную.
11. Когда условие финала выполнено, показывается итоговый экран с топ-3 и остальными игроками ниже.

## Скрипты

```bash
npm run install:all
npm run dev
npm run build
npm run start
```

## Переменные окружения

Сервер:

```text
PORT=3001
CLIENT_ORIGIN=http://localhost:5173
```

Клиент:

```text
VITE_SERVER_URL=http://localhost:3001
```

## Ограничения MVP

- Комнаты и очки хранятся в памяти процесса.
- При перезапуске сервера состояние сбрасывается.
- Если игрок отключился, он удаляется из комнаты.
- Если отключился хост, хостом становится следующий игрок.
- Если отключился объясняющий во время раунда, раунд завершается как `skipped`.
