# Слова-мины

Онлайн-игра для компании с комнатами, ролями, таймерами, минами и очками. Игроки общаются во внешнем созвоне, а сайт ведет игровое состояние.

## Как играть

1. Хост создает комнату, выбирает настройки и отправляет ссылку игрокам.
2. В каждом раунде сервер случайно выбирает объясняющего и угадывающего. Остальные становятся минерами.
3. Минеры заранее добавляют слова-мины. Объясняющий и угадывающий список мин не видят.
4. Мины можно редактировать до начала объяснения.
5. Когда все мины готовы или таймер на мины вышел, начинается объяснение.
6. Во время объяснения минеры отмечают сработавшие мины. Отметку можно снять при ошибочном клике.
7. Если слово угадали, объясняющий нажимает `Угадали`.
8. Угадывающий может нажать `Сдаться`, а объясняющий может завершить раунд как неугаданный.
9. За угаданное слово объясняющий и угадывающий получают по 1 очку.
10. За неугаданное слово или таймаут объясняющий и угадывающий теряют по 1 очку.
11. Каждая сработавшая мина дополнительно снимает по 1 очку с объясняющего и угадывающего, а автору мины дает +1.
12. После результата следующий раунд запускается автоматически, пока не выполнено условие финала.

## Возможности

- Комнаты по ссылке.
- Realtime-синхронизация через Socket.IO.
- Разное состояние для ролей: секретные мины не отправляются объясняющему и угадывающему.
- Случайный выбор пары объясняющий/угадывающий через перемешанные очереди.
- Настройки комнаты: сложность, количество мин, таймеры, финал по очкам или количеству раундов.
- Уровни сложности: легкий, средний, нереальный.
- Свой словарь для хоста: слова и фразы можно писать с новой строки или через запятую, хранится в памяти комнаты.
- Светлая и темная тема.
- Переключатель звука и звук последних секунд таймера.
- Пауза таймера для хоста.
- Серверная синхронизация таймера: сервер отправляет оставшееся время при смене состояния, клиент считает локально.
- Reconnect по ссылке комнаты и локальному токену игрока.
- Окно восстановления игрока после обрыва соединения: 15 минут.
- Комната не удаляется, если все игроки вышли. Она остается с тем же `roomId`, сохраняет настройки и словарь, но возвращается в лобби.
- Хост может кикать участников после подтверждения.
- Итоговый экран с топ-3.

## Стек

- Frontend: React, Vite, TypeScript.
- Backend: Node.js, Express, Socket.IO, TypeScript.
- Хранение: in-memory.

## Локальный запуск

```bash
npm install
npm run dev
```

По умолчанию:

- клиент: `http://localhost:5173`
- сервер: `http://localhost:3001`

## Production / Render

Проект рассчитан на деплой как один Node Web Service.

```text
Build Command: npm install && npm run build
Start Command: npm start
```

Environment:

```text
NODE_ENV=production
UPSTASH_REDIS_REST_URL=...
UPSTASH_REDIS_REST_TOKEN=...
```

Render задает `PORT` автоматически. В production сервер отдает собранный `client/dist`, а клиент подключается к Socket.IO на том же домене.

Redis-переменные необязательны для локальной разработки. Если они не заданы, комнаты работают только in-memory. Если заданы, комнаты сохраняются в Upstash Redis и восстанавливаются после рестарта Node-процесса.

## Ограничения

- Без Redis комнаты, очки, настройки и словари хранятся в памяти процесса.
- С Upstash Redis комнаты восстанавливаются после рестарта Node-процесса.
- WebSocket-соединения при рестарте все равно рвутся, но клиент автоматически переподключается и повторно входит в комнату по локальному токену игрока.
- Если Redis недоступен или не настроен, проект возвращается к in-memory режиму.
